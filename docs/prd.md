# 需求文档

## 1. 应用概述

### 1.1 应用名称
智能健康助手（AI营养秤小程序）

### 1.2 应用描述
基于微信小程序的智能健康管理工具，配套BLE营养秤硬件使用，集成百度多模态AI能力（食物识别、语音识别、智能对话），为用户提供实时称重、营养分析、AI健康咨询、饮食统计、家庭成员健康档案管理等功能。产品定位为健康管理工具，不涉及医疗诊断。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 关注健康饮食的个人用户
- 需要管理慢性病饮食的用户
- 有过敏源需规避的用户
- 需要管理家庭成员健康档案的用户

### 2.2 核心使用场景
- 日常烹饪前称重食材并获取营养分析
- 咨询AI获取个性化饮食建议
- 查看饮食统计数据并对比目标
- 管理家庭成员健康档案并切换使用

---

## 3. 页面结构与功能说明

### 3.1 页面结构

```
智能健康助手小程序
├── 底部导航
│   ├── 首页（称重&分析中枢）
│   ├── AI问答页（智能健康顾问）
│   ├── 统计页（饮食数据看板）
│   └── 我的（个人中心）
├── 设备管理页
├── 设备添加&配对流程页
├── 个人资料页（健康档案）
├── 家庭成员管理页
└── 菜谱推荐页
```

### 3.2 全局功能

#### 3.2.1 网络异常处理
- 断网时顶部Toast提示，加载区域显示骨架屏或空白状态页（含重试按钮）
- 云端AI功能禁用，本地缓存数据（历史记录、个人档案）仍可查看
- AI分析功能显示「需要网络连接」提示

#### 3.2.2 权限申请说明
- 首次使用相机、麦克风、蓝牙、位置功能时弹出权限申请说明弹窗
- 拒绝权限时功能降级：拒绝相机→仅支持手动输入；拒绝麦克风→仅支持文字/拍照；拒绝蓝牙→仅支持手动输入重量
- 每次弹窗说明该权限用途及影响

#### 3.2.3 底部导航
- 4个标签：首页 / AI问答 / 统计 / 我的
- 导航栏背景绿色，标题「智能健康助手」
- 标签切换保留页面滚动状态，AI问答、统计页切换时不中断正在进行的分析请求

#### 3.2.4 健康建议免责声明
- 首次启动展示全屏免责弹窗，用户须勾选「我已阅读并同意」后进入首页，不同意则无法使用
- 免责文案：「本应用提供的营养分析及健康建议仅供参考，不能替代专业医生或营养师诊断，请勿据此自行调整治疗方案」
- AI分析结果页、AI回答页、菜谱页底部固定显示免责小字

#### 3.2.5 过敏源实时预警
- 食材识别/AI分析结果中命中用户过敏源时，顶部显示红色高亮警告Banner
- 过敏源预警触发规则：AI识别食材名称与用户档案过敏源字段做字符串匹配+语义匹配
- 命中时在食材名旁显示⚠️标识，分析结果顶部显示红色警告「该食材可能含您过敏的成分：[过敏源名称]，请注意」
- 菜谱推荐时自动过滤含过敏食材的方案

#### 3.2.6 BLE连接状态常驻显示
- 首页顶部显示连接状态图标（已连接绿色/断开红色/连接中灰色动画）+ 电量百分比
- 设备断开连接时立即更新状态图标
- 断线时称重相关功能置灰，显示「请先连接营养秤」
- 电量低于20%时展示低电量警告

#### 3.2.7 断线自动重连
- 连接中断后自动尝试重连3次，间隔5秒
- 3次失败后停止自动重连，提示用户手动操作
- 用户进入首页时触发一次主动连接尝试

### 3.3 首页（称重&分析中枢）

#### 3.3.1 BLE实时称重显示
- 大字体卡片展示当前重量（带单位g/kg/lb）
- 稳定状态指示灯（稳定绿色/不稳定灰色闪烁）
- 去皮按钮
- 小程序启动后自动初始化蓝牙适配器并连接上次绑定设备
- 重量数据通过Notify UUID实时更新
- 连续1.5秒内重量变化≤0.5g判定为「稳定」，稳定后自动高亮显示当前读数
- 设备未连接时显示「--」并展示手动输入重量入口

#### 3.3.2 单位切换
- 称重卡片右上角单位切换按钮（g / kg / lb:oz）
- 点击切换单位 → 下发DPID 25命令至设备 → 设备回传对应单位重量
- 小程序侧同步更新显示单位，切换单位不清空已录入食材列表

#### 3.3.3 去皮功能
- 容器放上后点击去皮按钮
- 点击去皮 → 下发DPID 24命令 → 设备回传归零后的净重
- 去皮状态在称重卡片上标注「已去皮」

#### 3.3.4 食材录入
- **文字输入**：底部输入框输入食材名称，重量自动填入当前稳定读数；设备未连接时重量字段可手动输入；点击添加 → 食材加入列表；食材名称不可为空，重量须>0
- **语音输入**：麦克风按钮（长按录音，最长10秒）；录音状态动画；识别结果文本预览；长按麦克风开始录音 → 松开结束 → 音频上传至百度多模态平台语音识别 → 返回食材名称文本填入输入框；识别失败时提示「未能识别，请重试或手动输入」；识别结果用户可编辑后再添加
- **拍照识别**：相机图标按钮；识别Loading动画；识别结果确认弹窗（识别到的食材名+可编辑）；点击拍照 → 调用微信相机/相册 → 图片上传至百度多模态平台图像识别 → 返回食物名称列表；识别结果弹窗供用户确认或修改；识别失败时提示重新拍摄并给出拍摄建议（光线充足/正面拍摄）

#### 3.3.5 食材列表管理
- 已录入食材卡片列表：食材名称、重量、操作按钮（分析/菜谱/删除）
- 底部综合菜谱推荐按钮
- 列表支持上下滑动
- 单项「分析」跳转单食材营养分析结果
- 单项「菜谱」跳转单食材菜谱推荐
- 「删除」左滑或点击删除图标二次确认后移除
- 底部「综合菜谱推荐」对当前全部食材进行分析

#### 3.3.6 用餐人数设置
- +/-按钮，数字显示，范围1-20人
- 人数影响综合营养分析时的「人均」计算，默认1人
- 人数调整后分析结果中人均数据实时更新（已出结果时需重新触发分析）
- +/-超出范围时按钮置灰

#### 3.3.7 营养分析展示
- AI分析结果卡片：Markdown渲染（热量/蛋白质/脂肪/碳水等）
- 底部固定免责声明小字
- 「继续咨询」按钮
- 过敏预警Banner（命中时显示）
- 调用百度多模态平台AI分析 → 流式返回Markdown内容渐进渲染
- 分析中显示骨架屏Loading
- 分析失败提示「分析失败，请检查网络后重试」并保留食材列表
- 「继续咨询」携带本次食材+分析结果上下文跳转AI问答页

#### 3.3.8 称重历史
- 首页下拉区域或独立入口
- 历史卡片：时间/食材列表/总热量，最近20条
- 每次完成营养分析后自动保存至本地（weighing_records集合）
- 有网时同步云端
- 点击历史记录 → 查看完整分析结果
- 支持「继续咨询」跳转AI问答

### 3.4 AI问答页（智能健康顾问）

#### 3.4.1 多轮对话展示
- 气泡对话流：用户气泡右侧、AI气泡左侧
- AI消息支持Markdown渲染
- 底部固定免责声明小字
- 保留完整多轮上下文
- 页面上拉加载更早消息（分页加载）
- AI回复流式渲染
- AI回复中包含营养/医疗建议时自动附加免责提示

#### 3.4.2 多模态输入
- 文字输入框 + 语音按钮 + 图片上传按钮
- 文字直接发送
- 语音通过百度多模态平台语音识别转文字后发送，识别结果用户可编辑
- 图片上传至百度多模态平台，AI基于图片内容回答

#### 3.4.3 上下文携带
- 新建对话时上下文选项面板：「使用个人资料」「使用当前食材」开关
- 默认携带用户个人资料（健康档案）作为AI背景信息
- 从首页/历史记录跳转时自动携带对应食材+分析结果上下文
- 个人档案中的过敏源/慢性病/用药信息在传给AI时须结构化说明，AI根据此调整建议

#### 3.4.4 对话历史管理
- 左侧抽屉：历史对话列表（标题+时间），最多50个
- 新建对话按钮
- 删除对话（长按或右滑）
- 历史对话云端同步（chat_history集合）
- 无网时展示本地缓存
- 对话标题默认取第一条消息前20字
- 删除对话需二次确认

#### 3.4.5 AI问答安全限制
- AI不得给出「确诊」「停药」「换药」等医疗诊断性建议
- 涉及用药互作、疾病治疗的问题回复末尾自动附加「请咨询专业医生」
- 用户档案中「正在服用的药物」字段仅用于规避食药互作提示，不用于诊断

### 3.5 统计页（饮食数据看板）

#### 3.5.1 时间筛选
- 顶部标签栏：今日 / 本周 / 本月
- 切换时间粒度 → 重新请求对应统计数据
- 首次加载展示「今日」数据

#### 3.5.2 营养素摄入统计
- 折线/柱状图：热量趋势
- 环形图：蛋白质/脂肪/碳水占比
- 数值卡片：总热量/平均热量
- 数据来源：云端statistics集合（云函数getStatistics）
- 云端请求失败时降级展示本地缓存数据并标注「数据可能不是最新的」
- 无数据时展示空状态引导用户开始记录

#### 3.5.3 营养摄入与目标对比
- 每日热量目标进度条（目标值可在个人档案设置）
- 超出目标时进度条变红
- 目标值未设置时展示行业参考值（成年男性2000kcal/成年女性1600kcal）并提示「点击设置您的目标」

### 3.6 个人中心页（账户&设置总览）

#### 3.6.1 用户信息
- 顶部头像（微信头像）、昵称、当前绑定设备数
- 点击头像区域跳转个人资料页

#### 3.6.2 设备连接概览
- 当前连接设备名称、连接状态、电量
- 点击跳转设备管理页

#### 3.6.3 功能入口导航
- 设备管理 / 个人资料 / 家庭成员 / 数据导出 / 提醒设置 / 使用帮助 / 设置
- 每项带右箭头图标，点击跳转对应页面

#### 3.6.4 健康数据报告导出
- 导出范围选择（时间段）、导出格式（PDF）、导出按钮
- 调用云函数exportData → 生成PDF报告（含饮食记录/营养统计）→ 保存到微信文件或转发
- 敏感健康字段（慢性病/用药）默认不包含在导出报告中，用户可勾选包含
- 导出前提示数据安全提醒

#### 3.6.5 测量&饮食提醒
- 开关列表：早餐前提醒/午餐前提醒/晚餐前提醒/饮水提醒
- 时间选择器
- 基于微信订阅消息或本地通知实现
- 用户可分项开关，不强制订阅

### 3.7 设备管理页

#### 3.7.1 已绑定设备展示
- 设备卡片：设备名称/型号/连接状态/电量
- 连接/断开按钮
- 解绑按钮
- 支持多设备绑定
- 点击「连接」发起BLE连接
- 点击「断开」断开当前连接
- 「解绑」需二次确认弹窗（「解绑后需重新配对」）
- 连接中显示Loading状态

#### 3.7.2 设备控制
- 熄屏时间设置（下拉：30秒/45秒/1分钟/2分钟/3分钟/5分钟/10分钟）
- 屏幕亮度（低/中/高/最亮）
- 操作项通过DPID 26/27下发命令至设备
- 设备未连接时控制项置灰

#### 3.7.3 添加新设备入口
- 右上角「+」按钮
- 点击跳转设备添加页

### 3.8 设备添加&配对流程页

#### 3.8.1 设备类型选择
- 设备类型列表（当前仅AI营养秤）
- 图文说明
- 选择设备类型后进入扫描步骤

#### 3.8.2 周边设备扫描
- 扫描动画
- 发现设备列表（设备名/信号强度）
- 扫描进度条（15秒）
- 自动初始化蓝牙 → 按serviceUUID过滤扫描 → 15秒后自动停止
- 扫描期间可手动停止
- 列表实时更新
- 未发现设备时展示排查建议（设备是否开机/距离是否过远）

#### 3.8.3 选择并绑定设备
- 点击列表中设备卡片进行配对
- 配对进度弹窗
- 成功/失败状态页
- 点击设备 → 发起BLE连接 → 连接成功保存绑定关系至云数据库
- 连接失败时仍保留绑定关系（下次自动重连）并提示「连接暂时失败，已记住设备，将自动重连」

#### 3.8.4 首次绑定命名
- 设备昵称输入框（默认「我的营养秤」）
- 确认按钮
- 绑定成功后弹出命名页
- 昵称保存至云数据库
- 完成后跳转首页并展示新手引导

#### 3.8.5 首次使用教程
- 操作步骤气泡引导（4步：放食材→录入食材名→开始分析→查看结果）
- 可跳过
- 仅首次绑定成功后展示
- 用户主动跳过或完成后标记hasSeenGuide=true，不再展示

### 3.9 个人资料页（健康档案）

#### 3.9.1 基础健康信息
- 性别（单选：男/女）
- 年龄（数字输入）
- 身高（cm）
- 体重（kg）
- 生日（日期选择器）
- 血型（下拉：A/B/AB/O/特殊）
- 保存至app.globalData.personalProfile同步云数据库
- 字段变更后AI问答自动使用最新档案

#### 3.9.2 慢性病标签
- 多选标签组：高血压/糖尿病/高血脂/痛风/肾病/其他（可手动输入）
- 选择后AI分析结果中自动规避相关禁忌食材提示
- 显示提示「该信息仅用于个性化营养建议，不用于医疗诊断」

#### 3.9.3 过敏源标签
- 多选标签组：花生/海鲜/坚果/乳制品/蛋类/麸质/其他（可手动输入）
- 保存后触发全局过敏预警机制
- 页面提示「设置后，含过敏原食材将自动标红提示」

#### 3.9.4 正在服用的药物
- 自由文本输入框（可填写多种，换行分隔）
- 输入框上方提示文案：「填写后AI将在菜谱和营养建议中提示可能的食药互作，不用于诊断」
- 该字段内容不在导出报告中默认包含

#### 3.9.5 每日营养目标
- 每日热量目标（kcal输入）
- 蛋白质/脂肪/碳水目标（可选填）
- 保存后在统计页展示目标进度条
- 提供参考值建议（基于身高体重BMI计算推荐范围）

#### 3.9.6 数据清除
- 「清除所有健康数据」按钮（红色）
- 「清除对话记录」按钮
- 点击后展示二次确认弹窗并列出将删除的数据范围
- 清除全部健康数据后个人档案重置为空
- 云端数据同步删除
- 操作不可撤销，弹窗中明确说明

### 3.10 家庭成员管理页

#### 3.10.1 家庭成员档案列表
- 成员卡片：头像（可自定义）/昵称/年龄/当前选中状态
- 右上角「+」添加按钮
- 支持创建多个成员档案（上限10人）
- 「主用户」（微信账号关联者）不可删除
- 列表顶部显示当前使用中的成员档案

#### 3.10.2 切换当前成员
- 点击成员卡片切换
- 切换成功Toast提示「已切换至[成员昵称]的档案」
- 切换后首页营养分析、AI问答、统计数据全部基于所选成员档案
- 切换操作立即生效
- 顶部常驻显示当前成员昵称

#### 3.10.3 新建/编辑成员档案
- 表单页：昵称、头像（相册选取）、基础信息（年龄/性别/身高/体重）、慢性病/过敏源/用药
- 字段规则与个人资料页一致
- 每位成员独立存储健康档案，互不干扰
- 保存时同步至云数据库

#### 3.10.4 删除成员
- 成员卡片长按或编辑页底部「删除成员」按钮（红色）
- 删除需二次确认并说明「该成员所有历史记录将一并删除」
- 删除主用户档案不可操作，按钮置灰

### 3.11 菜谱推荐页

#### 3.11.1 AI生成菜谱
- 菜谱卡片：菜名、食材用量列表、步骤说明（Markdown）

- 流式渲染逐步展示
- 生成失败提示重试
- 结果自动检查是否含用户过敏源，命中时顶部红色警告

#### 3.11.2 重新生成
- 「换一个」按钮
- 点击重新请求AI生成新方案，旧结果替换

#### 3.11.3 分享菜谱

#### 3.11.4 继续咨询

- 携带当前菜谱内容+食材信息跳转AI问答页

## 4. 业务规则与逻辑

### 4.1 BLE连接与数据传输
- 小程序启动后自动初始化蓝牙适配器并连接上次绑定设备
- 重量数据通过Notify UUID实时更新
- 连续1.5秒内重量变化≤0.5g判定为「稳定」
- 连接中断后自动尝试重连3次，间隔5秒
- 3次失败后停止自动重连，提示用户手动操作
- 用户进入首页时触发一次主动连接尝试
- 单位切换通过DPID 25命令下发至设备
- 去皮通过DPID 24命令下发至设备
- 设备控制（熄屏时间/屏幕亮度）通过DPID 26/27下发命令

### 4.2 过敏源预警机制
- AI识别食材名称与用户档案过敏源字段做字符串匹配+语义匹配
- 命中时在食材名旁显示⚠️标识
- 分析结果顶部显示红色警告「该食材可能含您过敏的成分：[过敏源名称]，请注意」
- 菜谱推荐时自动过滤含过敏食材的方案
- AI问答回复提及过敏食材时橙色文字高亮 + 角标

### 4.3 AI分析与对话上下文
- 营养分析调用百度多模态平台AI分析，流式返回Markdown内容渐进渲染
- AI问答默认携带用户个人资料（健康档案）作为背景信息
- 从首页/历史记录跳转时自动携带对应食材+分析结果上下文
- 个人档案中的过敏源/慢性病/用药信息在传给AI时须结构化说明
- AI不得给出「确诊」「停药」「换药」等医疗诊断性建议
- 涉及用药互作、疾病治疗的问题回复末尾自动附加「请咨询专业医生」

### 4.4 家庭成员档案切换
- 切换成员后首页营养分析、AI问答、统计数据全部基于所选成员档案
- 每位成员独立存储健康档案，互不干扰
- 主用户（微信账号关联者）不可删除

### 4.5 数据存储与同步
- 称重历史保存至本地weighing_records集合，有网时同步云端
- 对话历史保存至云端chat_history集合，无网时展示本地缓存
- 个人档案保存至app.globalData.personalProfile同步云数据库
- 慢性病、过敏源、用药信息等敏感健康字段在云数据库中独立加密存储

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 网络断开 | 顶部Toast提示，云端AI功能禁用，本地缓存数据仍可查看，AI分析功能显示「需要网络连接」提示 |
| 权限拒绝 | 功能降级：拒绝相机→仅支持手动输入；拒绝麦克风→仅支持文字/拍照；拒绝蓝牙→仅支持手动输入重量 |
| BLE连接失败 | 自动重连3次（间隔5秒），3次失败后停止并提示用户手动操作 |
| 设备断线 | 称重相关功能置灰，显示「请先连接营养秤」，自动尝试重连 |
| 设备电量低 | 电量低于20%时展示低电量警告 |
| 语音识别失败 | 提示「未能识别，请重试或手动输入」，识别结果用户可编辑后再添加 |
| 图片识别失败 | 提示重新拍摄并给出拍摄建议（光线充足/正面拍摄） |
| AI分析失败 | 提示「分析失败，请检查网络后重试」并保留食材列表 |
| 云端统计数据请求失败 | 降级展示本地缓存数据并标注「数据可能不是最新的」 |
| 无统计数据 | 展示空状态引导用户开始记录 |
| 营养目标未设置 | 展示行业参考值（成年男性2000kcal/成年女性1600kcal）并提示「点击设置您的目标」 |
| 删除成员 | 二次确认并说明「该成员所有历史记录将一并删除」，主用户不可删除 |
| 清除健康数据 | 二次确认弹窗列出将删除的数据范围，操作不可撤销 |
| 导出报告 | 敏感健康字段默认不包含，用户可勾选包含，导出前提示数据安全提醒 |

## 6. 验收标准

1. 用户打开小程序，首次启动展示免责声明弹窗，勾选同意后进入首页
2. 用户在设备管理页添加新设备，扫描并绑定AI营养秤，连接成功后首页顶部显示绿色连接状态图标
3. 用户在首页放置食材，称重卡片实时显示重量并判定稳定状态，点击去皮按钮归零
4. 用户通过文字/语音/拍照方式录入食材名称，食材加入列表
5. 用户点击「分析」按钮，AI返回营养分析结果（Markdown渲染），底部显示免责声明
6. 用户点击「继续咨询」跳转AI问答页，携带食材+分析结果上下文，发送问题后AI流式返回回答
7. 用户在统计页查看今日/本周/本月营养摄入统计，折线图/柱状图/环形图正常展示
8. 用户在个人资料页填写过敏源（如花生），保存后首页识别含花生食材时顶部显示红色预警Banner
9. 用户在家庭成员管理页新建成员档案，切换成员后首页营养分析基于所选成员档案
10. 用户在个人中心页导出健康数据报告，生成PDF文件并保存到微信文件
